# Use Next.js, Rust Axum, and PostgreSQL

The repository currently contains planning documents rather than an implementation, and the production MVP will be built as a staff-only scheduling and lesson balance system. We will use Next.js for the frontend, Rust with Axum for the backend API, and PostgreSQL for persistence, accepting higher implementation complexity than a simpler full-stack Node application in exchange for strong typing, explicit backend boundaries, and the chosen long-term production stack.
